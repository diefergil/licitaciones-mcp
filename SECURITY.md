# Security Policy

## Reporting a Vulnerability

Please report vulnerabilities privately by emailing the maintainer listed on the GitHub profile for this repository. Do not open a public issue for suspected vulnerabilities.

Include:

- affected version or commit;
- steps to reproduce;
- expected impact;
- any safe proof of concept.

## Supported Versions

The `main` branch is the supported development line until the first tagged release.

## Security Expectations

- Do not commit API keys, tokens, cookies, private keys, database dumps, or downloaded source datasets.
- Use environment variables for runtime secrets.
- Keep dependency updates and security workflows green before merging.
